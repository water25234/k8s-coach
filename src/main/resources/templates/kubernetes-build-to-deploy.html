Build to Deploy

Java 程式如何變成 Kubernetes 裡的 Pod

Kubernetes 不會直接讀你的 Java 原始碼。部署前要先把 Spring Boot 網站做成 Docker image， 然後 Deployment YAML 再指定這個 image，讓 Kubernetes 建立 Pod 來執行 container。

一條主線先看懂

1

寫 Java / HTML / CSS

src/main/java 是 Spring Boot 程式，src/main/resources/templates 是 Thymeleaf 頁面，src/main/resources/static 是 CSS 和圖片。

2

Dockerfile 打包 application

docker build 會讀 Dockerfile。本專案的 Dockerfile 會先用 Maven build 出 Spring Boot JAR，再放進 Java 21 runtime image。

3

建立 image 並加上 tag

指令 docker build -t k8s-coach:0.1.0 . 會建立一個 Docker image，並把 k8s-coach:0.1.0 這個 tag 貼到它身上。

4

Deployment 指定 image

k8s/deployment.yaml 裡的 image: k8s-coach:0.1.0 會告訴 Kubernetes：Pod 裡的 container 要用這個 image 啟動。

5

Kubernetes 建立 Pod

replicas: 2 表示 Deployment 會維持兩個 Pod。每個 Pod 裡面都跑同一份 Spring Boot container。

6

Service 讓流量找到 Pod

k8s/service.yaml 不會 build image，也不會建立 app。它用 selector 找到 Pod，提供穩定入口。

本專案檔案怎麼接起來

Dockerfile

負責把專案打包成 Docker image。它處理 Java build 和 runtime image。

docker build -t k8s-coach:0.1.0 .
docker-compose.yml

給本機 Docker 測試使用。它可以 build 同一份 Dockerfile，並把 container 的 8080 對到本機 8080。

docker compose up --build
k8s/deployment.yaml

給 Kubernetes 使用。它不負責 build，而是指定已經存在的 image，然後建立 Pod。

image: k8s-coach:0.1.0
k8s/service.yaml

給 Kubernetes 網路使用。它負責把流量導到 Deployment 建出的 Pod。

selector.app: k8s-coach

第一次部署順序

  1. 1
    先 build image docker build -t k8s-coach:0.1.0 .
  2. 2
    建立 Namespace kubectl apply -f k8s/namespace.yaml
  3. 3
    建立 Deployment，讓 K8s 建 Pod kubectl apply -f k8s/deployment.yaml
  4. 4
    建立 Service，讓流量可以找到 Pod kubectl apply -f k8s/service.yaml
  5. 5
    把 Service 暫時接到本機 kubectl port-forward service/k8s-coach 8080:8080 -n k8s-coach

修改網站後為什麼要重 build

你改 Java、HTML、CSS 或圖片時，只是改了專案檔案。Kubernetes 裡正在跑的 Pod 不會自動看到這些檔案， 因為 Pod 是從舊 image 啟動的。要讓新內容進 K8s，必須重新 build image，確認 cluster 拿得到新版 image，再重啟 Deployment。

  1. 1
    重新產生 image docker build -t k8s-coach:0.1.0 .
  2. 2
    要求 Deployment 重啟 Pod kubectl rollout restart deployment/k8s-coach -n k8s-coach
  3. 3
    等待新版 Pod 就緒 kubectl rollout status deployment/k8s-coach -n k8s-coach

簡單記法：Dockerfile 負責做 image，deployment.yaml 負責叫 Kubernetes 用 image 跑 Pod。 rollout restart 只會重建 Pod；如果 cluster 仍然拿到舊 image，畫面還是會是舊的。
